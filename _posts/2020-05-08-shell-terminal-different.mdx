## Shell是什麼？
Shell 其實就是一種程式，所以它包含 **variables(變數)** 、 **conditionals(條件判斷)** 、 **loops(迴圈)** 、 **functions(函式)** ，因此在執行 Shell 指令時，就是在編寫 Shell 直譯的一段程式碼，所以只要我們寫出不符合 Shell 指令的東西就會顯示錯誤。

## 什麼時候是用到shell的契機？
當想執行一些視覺介面無法滿足我們需求的事時（比如刪掉看不見的東西、去不能去的地方、新增下載檔案，或是節省我們點擊滑鼠的次數XD），就需要 Shell 藉由 program 的方式來和電腦相互溝通，達成操作介面進行某些更核心的行為。

---

## 所以shell就是終端機(Terminal)嗎？
終於來到本篇的重點，介紹了 shell ，那他到底跟終端機有什麼關係？我們透過終端機「**輸入**」shell 指令，再由 shell「**執行**」這段指令，最後再透過終端機「**顯示**」出來。

就像在家裡，「媽媽」給「我」了一個去洗碗的指令，而「我」去做了洗碗這件事。這放在 shell 和終端機就像「shell」給了「終端機」一個洗碗指令，而「終端機」印出了洗碗這個指令的結果一樣。
他們之間有什麼相同嗎？其實是完全不相同的東西，他們唯一有的關係就是「**互相合作**」來達成我給予的任務。
